# 예약 도구 (codex-scheduler)

> ⚠️ **보존용(아카이브) — 가르치지 않음.** Codex **데스크톱 앱**에 내장 예약 **"예약됨"**(사이드바 → '채팅으로 만들기', 대화형)이 있어 이 도구를 대체한다. 코치 스킬·강의는 전부 네이티브 "예약됨"으로 안내한다. 이 스크립트는 **앱을 못 쓰고 CLI만 쓰는 환경의 폴백**으로만 남겨둔다. (실측: 2026-06-30)

Codex로 **"매일 9시 자동으로 ~"** 같은 예약 실행을 하게 해주는 작은 도구다.

## 왜 필요한가

- **Claude Code**는 예약(cron) 기능이 내장돼 있다 → 이 도구가 **필요 없다.**
- **Codex CLI에는 내장 예약이 없다.** `codex exec`(한 번 실행)만 있다.
  이 도구가 그 빈자리를 채운다: 켜져 있는 동안 예약표를 보고, 시각이 되면 `codex exec`를 대신 깨운다.

## 한계 (정직하게)

- **이 도구가 켜져 있는 동안만 돈다.** (Claude Code의 내장 예약도 앱이 켜져 있어야 도는 것과 같은 제약.)
  PC가 꺼져 있거나 이 도구를 안 켜두면 예약은 발동하지 않는다.
- 그래서 "PC와 도구가 켜져 있는 상태"가 전제다.

## 안전장치

무인 실행이라 **사람이 옆에서 위험한 행동을 막아줄 수 없다.** 그래서 도구가 모든 예약 지시 앞에
강제로 안전 문구를 붙인다: *읽기·정리·요약·초안 작성만 하고, 결제·주문확정·발송·삭제·등록·대량변경 등
되돌리기 어려운 행동은 하지 않는다.* 그런 게 필요하면 실행하지 않고 "사장님 확인 필요"로 기록만 남긴다.

> Windows에서는 Codex 샌드박스가 OS 기능에 기대기 때문에 무인 쓰기가 `--dangerously-bypass-approvals-and-sandbox`
> 모드로만 동작한다. 즉 예약 실행 중 Codex는 셀러 PC에서 제약 없이 동작할 수 있다. **자기 PC에서, 안전한 작업에만** 쓴다는 전제다.

## 쓰는 법

1. 예약표를 만든다(`schedule.example.json` 참고). 각 작업:
   - `id`: 이름(기록에 표시)
   - `cron`: 5필드 표준 cron — `분 시 일 월 요일` (예: `3 9 * * *` = 매일 9시 3분, `*/30 9-18 * * 1-5` = 평일 9~18시 30분마다)
   - `recurring`: `true`=반복, `false`=한 번만 하고 예약표에서 사라짐
   - `cwd`: AI가 일할 폴더(사업 폴더)
   - `prompt`: 시킬 일(안전한 작업만)
2. 도구를 켠다:
   ```
   node scheduler.js --file 예약표.json
   ```
   (켜 두면 시각에 맞춰 알아서 실행한다. 결과는 `logFile`(기본 `실행기록.md`)에 사람이 읽는 형태로 쌓인다.)

> PC를 켤 때마다 자동으로 이 도구가 켜지게 하려면 OS 자동시작 등록이 한 번 필요하다(다음 단계).
> 설치·시작은 보통 **환경 준비 도우미(setup-helper)**가 대신 해준다 — 셀러가 직접 명령을 칠 일은 없다.
